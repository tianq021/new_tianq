# FastGPT JSON 生成规则整改说明（maxFiles 与单/多文件逻辑）

整改时间：2026-05-11

## 1. 被影响的文件

主要影响文件有两个：

1. `FastGPT_工作流JSON生成通用规则_给AI模型使用_2026-05-11_源码约束修订版.md`
   - 问题：`chatConfig.fileSelectConfig` 示例里直接写了 `maxFiles: 5`。
   - 风险：AI 可能把 5 当成默认强规则，而不是示例值。

2. `FastGPT_单节点母版与工作流拼接规则_给AI模型使用_2026-05-11.md`
   - 问题：文件上传应用示例里直接写了 `maxFiles: 10`。
   - 风险：其他 AI 复制母版时可能默认允许多文件，但后续代码仍只处理 `files[0]`。

帮助总结类文件不是直接原因，只是建议增加错误案例库和自检清单。

## 2. 整改原则

`maxFiles` 不能直接照抄母版示例，必须和实际处理逻辑绑定。

- 单文件流程：`maxFiles = 1`。
- 多文件流程：`maxFiles > 1` 时，代码必须遍历 `files`，AI 输出必须是数组，例如 `receipts[]`，总结节点必须按多份文件汇总。
- 禁止出现：`maxFiles > 1`，但代码只读取 `files[0]`。

## 3. 已整改内容

1. 核心规则文件新增“文件上传数量规则（硬规则）”。
2. 单节点母版文件新增“文件上传数量规则（硬规则）”。
3. 单节点母版的文件上传示例默认改为 `maxFiles: 1`。
4. 自检清单新增：检查 `maxFiles` 是否与代码、AI 提示词、开场白、最终回复一致。
5. 常见错误新增：`maxFiles` 与实际处理逻辑不一致。
6. 当前票据识别 JSON 改为单文件稳定版：`maxFiles = 1`，开场白和错误提示同步改为“请上传 1 份”。

## 4. 后续使用建议

如果用户只是说“上传票据识别”，默认生成单文件版。

如果用户明确说“批量识别多份票据”，才生成多文件版，并且必须改成数组输出结构。
