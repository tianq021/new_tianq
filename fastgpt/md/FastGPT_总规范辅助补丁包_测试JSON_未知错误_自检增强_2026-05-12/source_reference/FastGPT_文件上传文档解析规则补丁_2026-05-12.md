# FastGPT 文件上传 + 文档解析工作流修正规则补丁（给 AI 模型使用）

## 问题来源
用户测试“上传文件 → 文档解析 → AI 总结 → 指定回复”工作流时发现：
- 文件上传开关虽然开启，但文档解析没有真正拿到上传文件；
- 文档解析节点引用了错误变量；
- 指定回复可能把数组引用当普通文本输出。

## 必须写入总规范的规则

### 1. 文件上传工作流必须同时满足三件事

#### A. workflowStart 必须有 userFiles 输出

```json
{
  "id": "userFiles",
  "key": "userFiles",
  "label": "app:workflow.user_file_input",
  "type": "static",
  "valueType": "arrayString"
}
```

#### B. chatConfig 必须开启 fileSelectConfig

```json
"fileSelectConfig": {
  "canSelectFile": true,
  "canSelectImg": false,
  "maxFiles": 10,
  "canSelectVideo": false,
  "canSelectAudio": false,
  "canSelectCustomFileExtension": false,
  "customFileExtensionList": []
}
```

#### C. readFiles.fileUrlList 必须引用 userFiles，不是 userChatInput

正确：

```json
"value": [["start", "userFiles"]]
```

错误：

```json
"value": [["start", "userChatInput"]]
```

### 2. readFiles 节点输入必须是二维数组引用

```json
{
  "key": "fileUrlList",
  "renderTypeList": ["reference"],
  "valueType": "arrayString",
  "required": true,
  "value": [["start", "userFiles"]]
}
```

### 3. 指定回复输出 AI 结果时，优先用文本插值

更稳写法：

```text
{{$aiChat.answerText$}}
```

风险写法：

```json
["aiChat", "answerText"]
```

如果用数组引用，必须确认平台导出里该输入处于 reference 模式；否则可能被当作普通文本显示。

### 4. 自检规则新增

生成文件上传工作流后，必须检查：
1. `workflowStart.outputs` 是否存在 `userFiles`；
2. `chatConfig.fileSelectConfig` 是否开启对应上传类型；
3. 所有 `fileUrlList` 是否引用 `userFiles`；
4. 不能把 `userChatInput` 当文件链接传给 `readFiles`；
5. `readFiles.system_text` 是否被后续文本拼接或 AI 节点引用；
6. 最终 `answerNode` 是否引用真实存在的 `aiChat.answerText`。
